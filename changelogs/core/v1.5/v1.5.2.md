# v1.5.1-v1.5.2

This changelog contains logs for v1.5.1-v1.5.2.

## UrlBuilder

This new tool allows creating and manipulating URI/URLs through Sisk. It is a modern alternative to [UriBuilder](https://learn.microsoft.com/en-us/dotnet/api/system.uribuilder).

```csharp
var uri = "http://example.com/path/to/resource?key1=value1&key2=value2";

var builder = new UrlBuilder(uri)
    .Pop()
    .AddSegment("index.html")
    .AddQuery("foo", "bar")
    .RemoveQuery("key2")
    .SetFragment("inner-section");

uri = builder.ToString();
// http://example.com/path/to/index.html?key1=value1&foo=bar#inner-section
```

## Other fixes and changes

- Changed the `HttpRequest` trace identifier to be generated on Sisk-side.
- **Breaking:** changed the `DefaultMessagePage` to be an instance instead of being static. This will likely require adaptation in your code.
- Http Engine abstraction fixes.
- Error corrections in `StringKeyStoreCollection`.
